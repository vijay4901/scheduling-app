# Contributing to Scheduling MVP

Thanks for your interest in contributing! This document provides guidelines for contributing to the project.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/your-username/scheduling-mvp.git`
3. Install dependencies: `npm install`
4. Create a branch: `git checkout -b feature/your-feature-name`

## Development Workflow

1. Make your changes
2. Test thoroughly
3. Commit with a descriptive message
4. Push to your fork
5. Create a Pull Request

## Code Style

- Use TypeScript for all new code
- Follow existing code structure
- Use Prettier for formatting
- Add comments for complex logic
- Keep functions small and focused

## Testing

Before submitting:

- [ ] Test all user flows manually
- [ ] Check console for errors
- [ ] Test on mobile viewport
- [ ] Verify database migrations work
- [ ] Test with different timezones

## Commit Messages

Use clear commit messages:

```
feat: Add email reminder feature
fix: Resolve timezone conversion bug
docs: Update README with deployment steps
refactor: Simplify booking creation logic
```

## Pull Request Process

1. Update README.md if needed
2. Describe your changes clearly
3. Link related issues
4. Wait for review
